## Convención de Commits

Utilizamos parte de [Conventional Commits](https://www.conventionalcommits.org/es/v1.0.0/) para el historial de versiones. El formato de los mensajes de commit debe ser:

`<tipo>: <descripción>`

### Tipos de commit permitidos:

* **feat**: Añade una nueva funcionalidad.
* **fix**: Soluciona un error (bug).
* **refactor**: Modifica el código sin añadir funcionalidades ni corregir errores (ej. renombrar variables, simplificar funciones).
* **chore**: Tareas de mantenimiento, cambios en la configuración o dependencias que no afectan el código fuente.
* **docs**: Cambios exclusivos en la documentación (ej. actualizar este README).
* **test**: Añade o corrige pruebas automáticas.

**Ejemplo:**
`feat: agregar autenticación con token jwt`
